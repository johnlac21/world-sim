1. UI Philosophy

World Sim BBGM inherits its design identity from Basketball GM:

Data-forward: tables, stats, and text-first presentation

Dense information layout: minimal whitespace, compact rows

Minimalistic visual design: avoid ornamentation, focus on clarity

Zero-friction navigation: side nav, top nav, consistent links

Fast scanning: bold titles, descriptive section headers, predictable positions

BBGM-like structure but NOT copying assets

Consistency over decoration ‚Äî every panel looks and behaves the same

Utility-first ‚Äî the player must feel like a ‚ÄúGM managing a country‚Äù through numbers, lists, and dashboards

This document defines the exact components, layouts, colors, and behaviors that the UI must follow.

3. Component Library (UI System)

All reusable UI components must follow this spec.

3.1 Panel / Card Component

Container for any BBGM-like block.

Style:

White background

Border: 1px solid #ddd

Border radius: 6px

Padding: 12px

Title font: 15px semi-bold

Spacing between title + content: 4px

Card nesting:

You MAY put panels inside panels, but avoid >2 levels deep.

3.2 Table Component

Core table appearance across standings, leaders, youth pipeline, performance lists.

Rules:

Font size: 13px

Row height: compact (28‚Äì32px)

Background: white

Odd rows: #fafafa

Hover row: #eef3ff

Borders: thin (#e3e3e3)

Numbers: right-aligned

Names: left-aligned

Sorting icons: up/down arrows, grey

Interaction:

Clicking row navigates if applicable

Hover shows pointer cursor

3.3 Depth Chart / Hierarchy Component

Used for:

Company hierarchy

Government offices

Specification:

One row per role

Left: role name

Middle: person name (clickable)

Right: stat-fit badge or simple colored circle

Locked icon on the far right (padlock outline)

Rows tightly spaced (28‚Äì30px)

Optional role color-coding by rank (light greyscale gradient)

3.4 Headline Card

Used in dashboard right column to show:

Visual spec:
