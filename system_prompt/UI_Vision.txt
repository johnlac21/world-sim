1. UI Philosophy

World Sim BBGM inherits its design identity from Basketball GM. The UI should feel like a single dense sheet of information, not a collection of floating cards.

Core principles:

Data-forward ‚Äì Tables, stats, and text-first presentation. Numbers and lists are the primary visual elements.

Dense layout ‚Äì Minimal whitespace, compact rows, small fonts. Everything important is visible at once.

Flat, single-sheet design ‚Äì One white content surface with sections and tables, not lots of rounded panels.

Zero-friction navigation ‚Äì Side nav + top nav, predictable locations for important links.

Fast scanning ‚Äì Strong page titles, small section headings, consistent ordering of sections.

BBGM-like, not BBGM-cloned ‚Äì Match structure, density, and flow; don‚Äôt copy assets or artwork.

Consistency > decoration ‚Äì A few well-defined patterns used everywhere.

Utility-first ‚Äì The user should feel like a GM managing a country via numbers, not like a casual app user.

This document defines how layouts, typography, and shared components work to support that philosophy.

3. Component Library (UI System)

We still use shared components, but cards/panels are no longer the default building block. The default look is flat sections + tables.

3.1 Flat Section Pattern (Default)

The canonical building block is:

A section heading

Optional subtext

One or more tables or short lists

Example:

Heading: ‚ÄúWorld snapshot‚Äù

Subtext: ‚ÄúKey indicators for Year 8‚Äù

Content: definition list, small stat blocks, buttons.

Style

Heading: 12px, semi-bold, normal case (NOT all caps)

Subtext: 11px, grey (#777)

Section spacing: 8‚Äì16px between sections

No surrounding card; background is the page sheet.

3.2 Panel / Card Component (Secondary)

Panels are special-purpose, not default layout. Use them sparingly when:

Content must pop from surrounding noise (e.g., headlines, alerts)

A bounded area makes the UI clearer (e.g., modal, popover, sidebox)

Style

Background: white

Border: 1px solid #ddd

Border radius: 4‚Äì6px

Padding: 8‚Äì12px

No heavy shadows (light or none)

Rules:

Avoid nesting panels (no >1 level deep)

Do not wrap whole pages in a grid of panels; rely on flat sections instead.

3.3 Table Component (Primary)

Tables are the core of the UI.

Appearance

Font size: 12px

Row height: compact (24‚Äì30px)

Background: white

Row striping: optional and subtle; single background is fine

Row border: 1px solid #eee or lighter

Header:

11px, uppercase or small caps

Bold or semi-bold

Bottom border stronger than row borders

Numbers right-aligned, names left-aligned

Interaction

Cell text links where appropriate (countries, companies, people)

Clicking a row may navigate to a detail page; if so, cursor should indicate clickability.

Optional sort by clicking headers (future)

Tables should feel exactly like BBGM: tight, informative, un-fancy.

5. Interaction Rules

All clickable names (people, companies, countries) are blue and underline on hover.

Table rows with row-level actions should change background on hover.

Up/down changes shown with green ‚Üë and red ‚Üì.

Sorting by clicking headers is ideal (can be rolled out gradually).

Modals and overlays use a simple, flat card ‚Äî same palette, no heavy shadows.

7. Non-Goals

We explicitly are not aiming for:

Fancy card grids everywhere

Large, airy, marketing-style layouts

Heavy animations or transitions

Portrait-heavy player cards (beyond small avatars)

Dark mode (for now)

Mobile-first layouts (desktop is primary; mobile comes later)

Goal: clarity, density, and a GM-simulator feel, not visual flash.
